## スライド 2: 解決したい課題

### 従来の学習ポートフォリオの問題点

- **主観的な記録**: ユーザーが自由に編集・装飾できる
- **改ざんリスク**: 後から記録を書き換え可能
- **証明困難**: 第三者が真正性を検証できない
- **信頼性の欠如**: 就職活動での説得力に欠ける

### 具体例
「私は100時間Reactを勉強しました」
→ **本当？** 証明できない

---

## スライド 3: ソリューション概要

### SymProof Tree の特徴

1. **ブロックチェーンで記録**
   - Symbolブロックチェーンに学習記録を永続保存
   - 改ざん不可能な証跡を作成

2. **即座に検証可能**
   - トランザクションハッシュで誰でも検証
   - 数秒で真正性を確認

3. **視覚的な進捗管理**
   - 学習ツリーで知識の体系を可視化
   - 学習の全体像を把握

4. **ポートフォリオ共有**
   - URLで簡単に共有
   - 企業に客観的な学習履歴を提示

## スライド 4: システムアーキテクチャ

### 技術構成

```
┌─────────────────────────────────┐
│     ブラウザ（Next.js App）      │
│                                 │
│  学習記録作成 → LocalStorage    │
│       ↓                         │
│  SSS Extension（署名処理）       │
└─────────────────────────────────┘
           ↓
┌─────────────────────────────────┐
│   Symbol Blockchain Network     │
│   - TransferTransaction         │
│   - 改ざん不可能な記録           │
│   - ブロック承認（約30秒）       │
└─────────────────────────────────┘
           ↓
┌─────────────────────────────────┐
│   誰でも検証可能                 │
│   - トランザクションハッシュ     │
│   - Symbol Explorer             │
└─────────────────────────────────┘
```

### 使用技術
- **フロントエンド**: Next.js 14 + React 18 + TypeScript
- **UI**: Atlaskit Design System（Atlassian製）
- **ブロックチェーン**: Symbol SDK v3 + SSS Extension
- **可視化**: React Flow（学習ツリー）
- **ストレージ**: LocalStorage + Symbol Blockchain

## スライド 8: 技術的な特徴

### ブロックチェーン連携の仕組み

#### トランザクション構造
```json
{
  "type": "TransferTransaction",
  "recipient": "[ユーザー自身のアドレス]",
  "mosaics": [],
  "message": {
    "type": "PlainMessage",
    "payload": "{学習記録のJSON}"
  }
}
```

#### メッセージペイロード例
```json
{
  "title": "React Hooksの学習",
  "subject": "プログラミング",
  "startTime": "2025-01-13T10:00:00Z",
  "endTime": "2025-01-13T11:30:00Z",
  "duration": 90,
  "understanding": 4,
  "notes": "useState, useEffect, useContextを理解"
}
```

### セキュリティ
- **改ざん不可能**: ブロックチェーンに記録後は変更不可
- **透明性**: 誰でもトランザクションを検証可能
- **プライバシー**: 機密情報は記録しない設計

## スライド 10: 実装の課題と解決

### 課題1: Deadline計算の誤り
**問題**: トランザクションが `Failure_Core_Insufficient_Balance` で失敗
**原因**: Deadline値の計算ミス（通常のNumber型で計算後にBigIntへ変換）
**解決**: BigIntで直接計算
```typescript
// 修正前
const deadline = BigInt(symbolTime + hours * 60 * 60 * 1000);

// 修正後
const deadline = BigInt(symbolTime) + BigInt(hours * 60 * 60 * 1000);
```

### 課題2: 手数料設定
**問題**: 100 XYMという高額な手数料設定
**原因**: 初期設定のミス
**解決**: 1 XYM（1,000,000 μXYM）に変更

### 課題3: Atlaskit型エラー
**問題**: xcssのfont/media queryプロパティで型エラー
**解決**: inline stylesとstyled-jsxの併用
